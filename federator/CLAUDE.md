# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

`federator` is the GraphQL BFF in the `blogapi.miyamo.today` monorepo (siblings: `article-service`, `blogging-event-service`, `tag-service`, `read-model-updater`, `core`). It exposes a single `/query` GraphQL endpoint on Echo, fans queries out to the read-side Connect-RPC services (`article-service`, `tag-service`) and mutations out to the write-side (`blogging-event-service`) — it owns no database. Authentication is JWT via Amazon Cognito; `UploadImage` is proxied as a **client-streaming** Connect RPC to `blogging-event-service`.

## Common commands

Run from `federator/`.

```bash
# Build (CI runs `go mod tidy` first, then `go build -v ./...`)
go build ./...

# Lint — CI uses staticcheck (honnef.co/go/tools/cmd/staticcheck@latest)
staticcheck ./...

# Full test run — matches CI's package filter (excludes generated code + DI)
go test $(go list ./... | grep -v "mock" | grep -v "infra/fw/gqlgen" | grep -v "configs" | grep -v "infra/grpc") -v

# Single test / suite
go test ./internal/app/usecase -run TestArticle_Execute -v
go test ./internal/if-adapter/controller/graphql/resolver -run TestQueryResolver_Articles -v

# Run server locally (loads .env via godotenv; default PORT=8080)
go run ./cmd/main.go

# Regenerate mockgen mocks under internal/mock/** (mockgen must be on PATH)
go generate ./...

# Regenerate GraphQL server + models from .api/**/*.graphqls (gqlgen.yml)
go generate -tags gqlgen ./...     # see gqlgen.go; invokes `gqlgen generate`

# Regenerate protobuf/Connect client stubs from .proto/** (buf.gen.yaml)
go generate -tags protogen ./...   # see protogen.go; invokes `buf generate`

# Regenerate Wire DI after editing wire.go or provider/*
go run github.com/google/wire/cmd/wire ./internal/configs/di
```

Required runtime env: `ARTICLE_SERVICE_ADDRESS`, `TAG_SERVICE_ADDRESS`, `BLOGGING_EVENT_SERVICE_ADDRESS` (all `host:port`, used with `http://` + `connect.WithGRPC()`), `AWS_REGION`, `COGNITO_APP_CLIENT_ID`, `COGNITO_USER_POOL_ID`, `NEW_RELIC_CONFIG_APP_NAME`, `NEW_RELIC_CONFIG_LICENSE`. Optional: `PORT`.

Both `.api/` (GraphQL schema `blogschema`) and `.proto/` (`proto.miyamo.today`) are git submodules — run `git submodule update --init --recursive` from the repo root before regenerating.

## Architecture

Clean Architecture with Wire-driven DI. Federator has **no persistence layer**; "infra" for this service means the downstream Connect clients and the Cognito JWT verifier. When adding a new resolver field, all four layers must be updated together — the compile-time `var _ I = (*Impl)(nil)` checks in `internal/configs/di/provider/usecase.go` and `provider/presenter.go` will break the build if any binding is missing.

Request flow (illustrated by `Query.articles`):

1. **Resolver** — `internal/if-adapter/controller/graphql/resolver/*.resolvers.go` is gqlgen-generated from `.api/**/*.graphqls` (schema config in `gqlgen.yml`). Each resolver opens a New Relic segment, builds an `in-DTO` from the GraphQL args, calls the usecase via `r.usecases.<name>.Execute(ctx, in)`, then runs the out-DTO through `r.converters.<name>.To<Model>` and returns a `model.*` from `internal/if-adapter/presenters/graphql/model/models_gen.go`. The query/mutation `.resolvers.go` files are regenerated by gqlgen and preserve hand-written bodies — keep imports and helper funcs separate (e.g. in `errors.go`) so they survive regeneration.
2. **Usecase** — `internal/app/usecase/*.go`. Each usecase wraps exactly one downstream Connect call (`u.articleServiceClient.GetArticleById`, `u.tagServiceClient.GetAllTags`, `u.bloggingEventServiceClient.CreateArticle`, …), maps the protobuf response into a DTO (`dto.NewArticleTag`, etc.), and returns an out-DTO. `UploadImage.Execute` opens a client stream, sends one `Meta` frame followed by `chunkSize`-sized `data` frames, then calls `CloseAndReceive`. Timestamps from protobuf are converted with `synchro.In[tz.UTC](ts.AsTime())`; URLs go through `net/url.Parse`.
3. **Interface port** — `internal/if-adapter/controller/graphql/resolver/usecase/*.go` declares the usecase interfaces (`Article`, `Articles`, `Tag`, `Tags`, `CreateArticle`, `UpdateArticleTitle`, `UpdateArticleBody`, `UpdateArticleThumbnail`, `AttachTags`, `DetachTags`, `UploadImage`); `resolver/presenter/converters/converter.go` declares the converter interfaces. Concrete implementations live in `internal/app/usecase/` and `internal/if-adapter/presenters/graphql/converters/` (all converter methods hang off a single `Converter` struct which is bound to **every** converter interface in `provider/presenter.go`).
4. **Infra** — `internal/infra/grpc/{article,tag,blogging_event}/**` is buf-generated Connect client code (with `*connect` subpackages for the handler interface). `internal/infra/aws/cognito/cognito.go` implements `middlewares.Verifier` against Cognito JWKS (fetched per-request — the `// OPTIMIZE: enable cache` comment marks this as known tech debt). `internal/infra/fw/gqlgen/gqlgen.go` is the gqlgen executable-schema file.

**Resolver wiring via functional options.** `resolver.NewResolver(*Usecases, *Converters)` takes two option-built structs. `provider/gqlgen.go`'s `Usecases(...)` / `Converters(...)` constructors call `resolver.NewUsecases(WithArticleUsecase(...), …)` / `resolver.NewConverters(WithArticleConverter(...), …)`. When adding a new usecase/converter: (a) declare the interface in `resolver/usecase` or `resolver/presenter/converters`, (b) add the struct field + `With*` option in `resolver/resolver.go`, (c) implement it in `internal/app/usecase/` or `internal/if-adapter/presenters/graphql/converters/converter.go`, (d) register the binding in `provider/usecase.go` or `provider/presenter.go` and thread the dependency through `Usecases(...)` / `Converters(...)`.

**Echo / gqlgen server setup.** `provider/echo.go`'s `Echo` mounts the gqlgen `*handler.Server` at `POST /query` behind this middleware chain (order matters): `echo.WrapHandler(srv)` → `nrecho.Middleware(nr)` → `middlewares.SetLoggerToContext(nr)` → `middlewares.RequestLog()` → `middlewares.Auth(verifier)`. The `/playground` route is also auth-gated. `/health` returns 200 without auth. `GET /*` serves the `go-import` / `go-source` meta tags from the embedded `remote_import_paths.html` so the module is importable as `blogapi.miyamo.today/federator`. `provider/gqlgen.go`'s `GqlgenServer` enables WebSocket + multipart (`3<<30` = 3 GiB upload cap for `UploadImage`), LRU query cache, Introspection, APQ, and registers gqlgen middleware from `blogapi.miyamo.today/core/graphql/middleware` (`StartNewRelicTransaction`, `SetBlogAPIContextToContext`, `StartNewRelicSegment` around root fields).

**GraphQL scalar binding.** `gqlgen.yml` maps `DateTime` → `internal/pkg/gqlscalar.UTC` (wrapping `synchro.Time[tz.UTC]`), `URL` → `internal/pkg/gqlscalar.URL`, and `Markdown` → `graphql.String`. The `ID` scalar is bound to multiple gqlgen types so integer and string IDs both unmarshal. Do not hand-edit `internal/infra/fw/gqlgen/gqlgen.go` or `internal/if-adapter/presenters/graphql/model/models_gen.go` — edit the schema under `.api/` and rerun gqlgen.

**Observability & errors.** Every resolver, usecase, and converter method wraps its body in `defer newrelic.FromContext(ctx).StartSegment(...).End()`. The logger is pulled from context via `altnrslog.FromContext` (installed by `middlewares.SetLoggerToContext`); fall back to `log.DefaultLogger()` when absent. Use `github.com/cockroachdb/errors` (`WithStack`, `Is`, `WithMessage`, `Join`) — not stdlib `errors` or `pkg/errors` — so stack traces survive `nrpkgerrors.Wrap`. Resolvers use the package-local `ErrorWithStack` helper from `resolver/errors.go` and the sentinel `ErrFailedToConvertTo*` errors for converter-bool-false cases. The Echo `HTTPErrorHandler` funnels every error through `nrpkgerrors.Wrap` + structured logging before delegating to the default handler. JSON uses `goccy/go-json` via `core/echo/s11n.JSONSerializer`.

## Generated code (do not hand-edit)

CI excludes these paths from tests:

- `internal/infra/grpc/**` — buf-generated from the `.proto/` git submodule. `buf.gen.yaml` pins `go_package_prefix` to `blogapi.miyamo.today/federator/internal/infra/grpc`.
- `internal/infra/fw/gqlgen/gqlgen.go` and `internal/if-adapter/presenters/graphql/model/models_gen.go` — gqlgen output, configured by `gqlgen.yml`. Only the `*.resolvers.go` files are hand-edited; gqlgen preserves their bodies across regeneration.
- `internal/configs/di/wire_gen.go` — Wire output. Hand-edit `wire.go` (build tag `wireinject`) and the provider sets in `internal/configs/di/provider/`, then rerun Wire.
- `internal/mock/**` — `mockgen` output. Each source interface (usecase, converter, each `*connect` client) declares a `//go:generate mockgen -source=$GOFILE -destination=...` directive; run `go generate ./...` to refresh after editing an interface.

## Testing conventions

Federator uses **`go.uber.org/mock`** (gomock) with MockGen-generated mocks under `internal/mock/`, matching `blogging-event-service` (not the `ovechkin-dm/mockio` style used by `article-service`/`tag-service`). Table-driven tests (`tests := map[string]testCase{...}` + `t.Run(name, ...)`) are the dominant style; install the BlogAPI context with `blogapictx.StoreToContext(ctx, blogapictx.New(...))` before calling a resolver or usecase. Use `google/go-cmp` for DTO comparisons and construct timestamps via `synchro.New[tz.UTC](...)`.

## Deployment

`.build/package/Dockerfile` targets `linux/arm64` (Go 1.25.1 alpine, two-stage build, `-ldflags="-s -w" -trimpath`). CI runs on `ubuntu-24.04-arm` via the shared composite action `.github/custom_actions/ci/action.yaml`; deploy is triggered by `.github/workflows/deploy_federator.yaml` on pushes to `main` under `federator/**`. Keep new code ARM64-compatible and avoid cgo-only deps.
